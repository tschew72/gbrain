# Hot-reload (SIGUSR1) on the OpenClaw gateway does NOT retroactively switch model

Hot-reload (SIGUSR1) on the OpenClaw gateway does NOT retroactively switch model for running sessions — it only affects new sessions. To switch a running session's model mid-flight, use session_status with the model parameter. Verified 2026-06-18 10:25 SGT during the 2026.6.8 upgrade — gateway restart was clean but the active session stayed on minimax/MiniMax-M2.7 until session_status override forced it to claude-cli/claude-sonnet-4-6.
