# GBrain Lint + Orphan Resolution

## Root Causes

### Lint issues (267)
1. **Missing `created` field (262):** Auto-ingested pages via `mcp:put_page` don't always populate `created` in frontmatter, but `ingested_at` is reliable. Fix script reads `ingested_at` → `enriched_at` → file mtime as fallback.
2. **Missing `title` field (4):** Same as above — newer auto-stub pages skip frontmatter.
3. **No frontmatter (1, README.md):** Lowercase readme file was never normalized.
4. **Placeholder date (1):** Lint regex `\bYYYY-MM-DD\b` fires inside `<YYYY-MM-DD>` too — `<>` are non-word chars so the word boundary still applies.

### Orphan pages (204)
Two layered root causes:

1. **`DIR_PATTERN` whitelist (link-extraction.ts:86)** originally listed only `people|companies|meetings|concepts|...|entities`. Pages in `ops/`, `extracts/`, `decisions/`, `inbox/`, etc. could never be the TARGET of `[[ops/foo]]` wikilinks OR `[Foo](../ops/foo.md)` markdown links — both regexes require a `DIR_PATTERN`-whitelisted directory prefix. All orphans in these categories were invisible to the linker.
2. **130+ root-level stub pages** (e.g. `vooju.md`, `adria-scan.md`) auto-created by the project linker with NO directory prefix at all. These can't be linked via standard syntax because they fail `DIR_PATTERN` at the regex level.

## Lessons

1. **Lint `YYYY-MM-DD` regex catches `\b` boundaries even inside `<>` wrappers** — word boundaries fire on the `\b` between non-word and word characters, and `<`/`>` are non-word. Always substitute placeholder strings with the actual shell expression that will produce them.
2. **`yaml.safe_dump` adds `...` document-end markers** that confuse gbrain's frontmatter parser. Use single-quoted scalar strings and build frontmatter text manually.
3. **GBrain link extraction has TWO gates:**
   - `DIR_PATTERN` whitelist for standard `[[dir/slug]]` syntax
   - `link_resolution.global_basename` for bare `[[slug]]` syntax (Issue #972)
   Both must be enabled/extended to reach all pages.
4. **Project linker should write stubs WITH their canonical directory prefix** to avoid orphan creation. The 130+ root-level stubs are an outstanding cleanup item — they should be moved to `companies/`, `people/`, `projects/` etc. based on their `type` frontmatter.
5. **Compiled binary swap gotcha:** `/root/.bun/bin/gbrain.real` is a symlink to `/root/gbrain/src/cli.ts`, but the supervisor worker uses the COMPILED binary `/root/gbrain/bin/gbrain` via the `gbrain` CLI dispatcher. Patches to `src/` require both `bun run build` AND `pm2 restart gbrain-jobs-supervisor` to take effect.

- **Root-level stub cleanup:** Move `vooju.md`, `adria-scan.md`, `companies-bugcrowd.md`, etc. to their canonical directories based on `type: entity` frontmatter.
